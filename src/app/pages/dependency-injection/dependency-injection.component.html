Dependency Injection

Dependency Injection is the technique in which an object receives other objects that it depends on.

Dependency Injection is an extremely important topic to understand because it is the foundation for frameworks and testing. This is what you gain by understanding dependency injection:

  • It's easier to design classes for testing.
  • It's easier to learn Spring Boot / Angular.
  • It's easier to debug Spring Boot / Angular issues.

We are going to learn dependency injection by studying each keyword found within its definition.

  • Object
  • Dependency
  • Injection

By the end of this lesson, you should be able to understand what dependency injection is from a code standpoint and, you should be able to identify the different techniques used to inject a dependency. Let's get started!

What is an object?

In Java and Javascript, an object refers to a particular instance of a class, where the object can be a combination of variables, functions, and data structures.

To simplify this, let's use a metaphor. A class is a blueprint for how you want to define the states and behavior of an object. A blueprint is not a house. It's merely a piece of paper that describes how the house should be built. A class operates in the same manner.

For us to get a house, we must build it. You need a construction crew and raw materials. Once the house is completed, then you can call the finished product a house. In our analogy this is what's considered the object. It's a working creation defined by the blueprint (the class). Fortunately for us, a computer can instaneously create an object for us, so we don't have to worry about the manual labor! We still require raw resources, though, and that comes in the form of memory. An object resides in the memory of the computer and can be called upon within a program to execute a specific task. For example, if we had a Zombie object in memory for our shooter game, the Zombie object would probably hold important information such as hit points, location, sprite information, or inventory items. The game would need to obtain this information to determine how the zombie interacts with the player.
